# User Personas

## Persona 1: Sarah — The Patient

| Attribute | Detail |
|---|---|
| **Role** | Patient seeking triage |
| **Age Range** | 18-80 (broad demographic) |
| **Tech Comfort** | Moderate — comfortable with messaging apps and basic web portals |
| **Goals** | Get quick assessment of symptoms, understand urgency, see the right doctor without long wait times |
| **Pain Points** | Long phone hold times, inconsistent advice from different clinics, uncertainty about whether symptoms warrant ER visit |
| **Context** | Accesses system from home (web) or on-the-go (mobile), may be in pain or distress |
| **Key Need** | Fast, clear, reassuring guidance on next steps |

## Persona 2: Dr. Patel — The On-Call Physician

| Attribute | Detail |
|---|---|
| **Role** | On-call physician receiving emergency escalations |
| **Age Range** | 35-60 |
| **Tech Comfort** | High — uses multiple clinical systems daily |
| **Goals** | Receive only true emergencies, get complete patient context instantly, respond within protocol time |
| **Pain Points** | False-positive escalations that interrupt workflow, incomplete patient summaries requiring callbacks, delayed notification systems |
| **Context** | May be in clinic, at home, or handling another patient when escalation arrives |
| **Key Need** | Accurate urgency classification with complete, structured patient summary on first notification |

## Persona 3: Dr. Kim — The Specialist

| Attribute | Detail |
|---|---|
| **Role** | Specialist physician receiving routed patients |
| **Age Range** | 30-65 |
| **Tech Comfort** | High — primary interface is EHR system |
| **Goals** | Receive well-triaged patients with relevant clinical context, minimize redundant patient interviews |
| **Pain Points** | Patients routed to wrong specialty, incomplete referral information, having to re-assess from scratch |
| **Context** | Reviews incoming referrals in EHR between patient appointments |
| **Key Need** | Accurate routing with a complete SOAP note that includes triage reasoning |

## Persona 4: Maria — The Triage Nurse

| Attribute | Detail |
|---|---|
| **Role** | Registered nurse monitoring AI triage decisions and handling escalated ambiguous cases |
| **Age Range** | 28-55 |
| **Tech Comfort** | Moderate-High — trained on clinical systems |
| **Goals** | Ensure AI triage quality, handle cases the AI cannot resolve, maintain patient safety |
| **Pain Points** | Too many false escalations, insufficient context when AI hands off, no visibility into AI reasoning |
| **Context** | Monitors a dashboard of active triage sessions, intervenes when AI flags uncertainty |
| **Key Need** | Clear visibility into AI confidence level and conversation history when a case lands on her queue |

## Persona 5: James — The Clinic Administrator

| Attribute | Detail |
|---|---|
| **Role** | Administrator managing triage system configuration and compliance |
| **Age Range** | 35-55 |
| **Tech Comfort** | Moderate — comfortable with admin dashboards and reporting tools |
| **Goals** | Ensure system meets performance targets, maintain HIPAA compliance, manage specialist routing rules |
| **Pain Points** | Lack of visibility into triage volume and bottlenecks, manual compliance reporting, difficulty adjusting routing rules |
| **Context** | Reviews daily/weekly reports, adjusts system configuration, responds to compliance audits |
| **Key Need** | Operational dashboards, audit trails, and configuration management for routing and scheduling |
